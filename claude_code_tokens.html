fi

🎯 스마트 작업 습관으로 토큰 절약

  • 📐
    구체적인 프롬프트 — "코드 개선" ❌ → "auth.ts login 함수에 입력 검증 추가" ✅
  • 🗺️
    Plan 모드 활용 — Shift+Tab으로 구현 전 계획 수립 및 승인
  • ⏹️
    즉시 중단 — 잘못된 방향 시 Esc로 멈추고 /rewind로 복원
  • 🧪
    점진적 테스트 — 파일 하나씩 작성하고 테스트 후 진행
  • 🤖
    서브에이전트 위임 — 테스트/로그 처리는 서브에이전트에 위임, 요약만 수신
  • 🔌
    코드 인텔리전스 플러그인 — "Go to Definition"으로 grep + 파일 읽기 대체
  • ✅
    검증 타겟 제공 — 테스트 케이스나 예상 출력을 프롬프트에 포함
  • 🔄
    자동 최적화 — 프롬프트 캐싱, 자동 컴팩션이 내부적으로 비용 절감
